# Screens & Pages Overview

This document describes the main screens and pages available in the system, organized by user role. Since we cannot run the application to capture screenshots, this document also includes a screenshot plan for when screenshots become available.

---

## Login & Authentication

### Login Page
**Purpose**: Allows users to access the system securely with their username and password.

**Main Actions Available**:
- Enter username/email and password
- Log in to access the system
- View error messages if login fails

**Who Accesses It**: Everyone must log in to use the system.

**Screenshot Plan**:
- Show the login form with username/password fields
- Demonstrate the clean, professional interface
- Important because: This is the first impression of the system

## Summary

The system has approximately 50+ distinct screens organized by functionality and user role. Each screen is designed to support specific workflows and tasks. The interface is consistent across all screens, making it easy for users to navigate and learn the system.

All screens include proper access controls, so users only see screens appropriate for their role. The system automatically redirects users who try to access unauthorized pages.
