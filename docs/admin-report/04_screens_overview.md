# Screens & Pages (With Screenshot Plan)

## Core Screens

### Dashboard
- **Purpose:** Quick overview of key activity and metrics.
- **Main actions:** Review summaries and recent activity.
- **Roles:** All signed‑in users (admins see the most detail).

### Admin Dashboard
- **Purpose:** System‑wide overview for leadership and system health.
- **Main actions:** Review counts, attendance summary, and recent activity.
- **Roles:** Administrator.

### Analytics
- **Purpose:** High‑level insights and trends.
- **Main actions:** Review system analytics.
- **Roles:** Administrator.

### Students
- **Purpose:** Central list of student records.
- **Main actions:** Search, view, and update student profiles.
- **Roles:** Administrator, Registrar.

### Courses
- **Purpose:** Course catalog for programs.
- **Main actions:** Create and manage courses.
- **Roles:** Administrator, Registrar, Faculty.

### Sections
- **Purpose:** Class sections tied to courses.
- **Main actions:** Create sections and assign faculty.
- **Roles:** Administrator, Registrar, Faculty.

### Programs
- **Purpose:** Define academic programs.
- **Main actions:** Create and manage programs.
- **Roles:** Administrator, Registrar.

### Batches
- **Purpose:** Group students by intake.
- **Main actions:** Create and manage batches.
- **Roles:** Administrator, Registrar.

### Academic Periods
- **Purpose:** Define terms or semesters.
- **Main actions:** Create and manage academic periods.
- **Roles:** Administrator, Registrar.

### Groups
- **Purpose:** Organize students into groups.
- **Main actions:** Create and manage student groups.
- **Roles:** Administrator, Registrar.

### Departments
- **Purpose:** Manage academic departments.
- **Main actions:** Create and manage departments.
- **Roles:** Administrator, Registrar.

### Timetable
- **Purpose:** View and manage class schedules.
- **Main actions:** Review timetable and schedules.
- **Roles:** Administrator, Registrar, Faculty, Coordinator.

### Bulk Enrollment
- **Purpose:** Enroll many students at once.
- **Main actions:** Upload or enter bulk enrollment data.
- **Roles:** Administrator, Registrar.

### Attendance
- **Purpose:** Record class attendance.
- **Main actions:** Mark attendance and review records.
- **Roles:** Administrator, Faculty.

### Bulk Attendance
- **Purpose:** Record attendance for many students quickly.
- **Main actions:** Enter bulk attendance data.
- **Roles:** Administrator, Faculty.

### Eligibility Report
- **Purpose:** Check attendance eligibility.
- **Main actions:** Review eligibility results.
- **Roles:** Administrator, Registrar.

### Assessments
- **Purpose:** Create and manage assessments.
- **Main actions:** Define assessments and record scores.
- **Roles:** Administrator, Faculty.

### Gradebook
- **Purpose:** View grades for classes and students.
- **Main actions:** Review grade summaries.
- **Roles:** Administrator, Faculty, Student.

### Results
- **Purpose:** View official results.
- **Main actions:** Review results and status.
- **Roles:** Administrator, Faculty, Student, Exam Cell.

### Publish Results
- **Purpose:** Official release of results.
- **Main actions:** Publish and finalize results.
- **Roles:** Administrator, Exam Cell.

### Finance Dashboard
- **Purpose:** Overview of financial activity.
- **Main actions:** Review fees and payments status.
- **Roles:** Administrator, Finance.

### Fee Plans
- **Purpose:** Define tuition and other fees.
- **Main actions:** Create and update fee plans.
- **Roles:** Administrator, Finance.

### Voucher Generation
- **Purpose:** Issue payment vouchers to students.
- **Main actions:** Generate vouchers for terms or groups.
- **Roles:** Administrator, Finance.

### Vouchers List
- **Purpose:** Track issued vouchers.
- **Main actions:** Search and review voucher status.
- **Roles:** Administrator, Finance.

### Payments
- **Purpose:** Record and verify payments.
- **Main actions:** Enter payments and confirm receipt.
- **Roles:** Administrator, Finance.

### Collection Report
- **Purpose:** Review collected fees.
- **Main actions:** View collection summaries.
- **Roles:** Administrator, Finance.

### Defaulters Report
- **Purpose:** Identify unpaid accounts.
- **Main actions:** Review defaulters list.
- **Roles:** Administrator, Finance.

### Aging Report
- **Purpose:** Track how long fees have been outstanding.
- **Main actions:** Review aging categories.
- **Roles:** Administrator, Finance.

### Student Statement
- **Purpose:** Full fee statement for a student.
- **Main actions:** Review fee history.
- **Roles:** Administrator, Finance, Student.

### My Fees
- **Purpose:** Student view of their own fees.
- **Main actions:** Review personal fee status.
- **Roles:** Student.

### Users
- **Purpose:** Manage user accounts and access.
- **Main actions:** Create users, update roles, reset passwords.
- **Roles:** Administrator.

### Roles & Permissions
- **Purpose:** Define what each role can access.
- **Main actions:** Review role access rules.
- **Roles:** Administrator.

### Audit Logs
- **Purpose:** Record of key actions (who did what and when).
- **Main actions:** Review activity history.
- **Roles:** Administrator.

### Student Import
- **Purpose:** Add many students at once.
- **Main actions:** Import student data.
- **Roles:** Administrator, Coordinator.

### Admin Settings
- **Purpose:** Adjust system options without technical work.
- **Main actions:** Turn features on or off and set rules.
- **Roles:** Administrator.

### My Profile
- **Purpose:** View and update personal account details.
- **Main actions:** Update personal information and sign out.
- **Roles:** All signed‑in users.

## Screenshot Plan (When the System Is Run)

1. **Admin Dashboard** — Show leadership‑level counts and recent activity to illustrate oversight.
2. **Students List** — Demonstrate how student records are centralized and searchable.
3. **Enrollment or Bulk Enrollment** — Show how large intakes are handled efficiently.
4. **Attendance Screen** — Show daily class attendance entry and summaries.
5. **Results / Publish Results** — Show the path from grading to official release.
6. **Finance Dashboard** — Show fee status and overall collections.
7. **Payments or Vouchers List** — Show how fees are issued and tracked.
8. **Users Management** — Show how accounts and access are controlled.
9. **Admin Settings** — Show the ability to change key rules and toggles.
10. **Student View (My Fees / Gradebook)** — Show what students can see for transparency.
