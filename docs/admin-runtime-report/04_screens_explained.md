# Screen-by-Screen Explanation for Administrators

**Date:** January 10, 2026  
**Audience:** Non-technical administrators and staff

---

## Overview

This document explains each screen in the FMU Platform system in simple, non-technical language. Each screen is described from the perspective of an administrator who needs to understand what the screen does and who uses it.

---

## 1. Login Screen

**Screen Name:** Login Page  
**Purpose:** Entry point to the system - where users sign in  
**Who Uses It:** Everyone (administrators, faculty, students, registrars)  
**Key Actions Available:**
- Enter username or email address
- Enter password
- Click "Sign In" to access the system
- Check "Remember me" to stay logged in
- Click "Forgot password?" if password is forgotten

**What You Need to Know:**
- All users must log in before accessing the system
- Each user has their own username and password
- The system remembers who is logged in
- After successful login, users are taken to their dashboard

**Screenshot Reference:** `01_login.png`

---

## 2. Main Dashboard

**Screen Name:** Main Dashboard  
**Purpose:** Central hub showing overview after login  
**Who Uses It:** Everyone after they log in  
**Key Actions Available:**
- View quick information and statistics
- Navigate to different parts of the system using the left menu
- Access role-specific features
- View notifications or announcements

**What You Need to Know:**
- This is the first screen after login
- The left menu shows all available features based on your role
- Different users see different options (admins see more than students)
- You can always return here by clicking "Dashboard" in the menu

**Screenshot Reference:** `02_main_dashboard.png`

---

## 3. Admin Dashboard

**Screen Name:** Admin Dashboard  
**Purpose:** Administrative overview with system statistics and quick actions  
**Who Uses It:** System administrators only  
**Key Actions Available:**
- View system-wide statistics (number of students, courses, etc.)
- See recent activity and system information
- Access administrative tools quickly
- Monitor system health and status

**What You Need to Know:**
- Only administrators can see this screen
- Provides a high-level view of the entire system
- Shows important numbers at a glance
- Useful for daily system monitoring

**Screenshot Reference:** `03_admin_dashboard.png`

## 4. Students Listing

**Screen Name:** Students Page  
**Purpose:** View and manage all student records  
**Who Uses It:** Administrators and registrars  
**Key Actions Available:**
- View list of all students
- Search for specific students
- Filter students by various criteria (program, batch, status, etc.)
- Create new student records
- View student details
- Edit student information

**What You Need to Know:**
- This is the master list of all students in the system
- You can find any student quickly using search
- New students are added from this page
- Student information can be updated here

**Screenshot Reference:** `04_students_list.png`

## 5. Courses Listing

**Screen Name:** Courses Page  
**Purpose:** Manage the catalog of courses offered  
**Who Uses It:** Administrators, registrars, and faculty  
**Key Actions Available:**
- View all courses in the system
- Search for specific courses
- Filter courses by program or department
- Create new courses
- Edit course information
- View course details

**What You Need to Know:**
- This is the master list of all courses
- Courses are organized by program and department

- Administrators can manage all courses

## 6. Attendance Dashboard

**Screen Name:** Attendance Management  
**Purpose:** Record and track student attendance  
**Who Uses It:** Faculty and administrators  
**Key Actions Available:**
- View attendance records
- Record attendance for classes
- Generate attendance reports
- View attendance statistics
- Mark students as present or absent

**What You Need to Know:**
- Faculty use this daily to record attendance
- Attendance can be recorded for individual classes
- Reports show attendance trends over time
- Important for tracking student participation

## 7. Finance Dashboard

**Screen Name:** Finance Management  
**Purpose:** Manage fees, payments, and financial records  
**Who Uses It:** Finance staff and administrators  
**Key Actions Available:**
- View financial overview
- Manage fee plans and structures
- Generate payment vouchers
- Track payments received
- Generate financial reports (defaulters, collections, etc.)
- View student financial statements

**What You Need to Know:**
- Central location for all financial activities
- Fee plans determine what students pay
- Vouchers are generated for fee collection
- Reports help track payments and outstanding amounts

## 8. User Management

**Screen Name:** Admin Users  
**Purpose:** Manage all user accounts in the system  
**Who Uses It:** System administrators only  
**Key Actions Available:**
- View list of all users (admin, faculty, students, etc.)
- Create new user accounts
- Edit user information
- Reset user passwords
- Activate or deactivate user accounts
- Assign user roles and permissions

**What You Need to Know:**
- Critical for system administration
- Only admins can create and manage users
- Users must be active to log in
- Roles determine what users can see and do
- Last admin account cannot be deactivated (safety feature)

**Screenshot Reference:** `08_admin_users.png`

## 9. Academic Programs

**Screen Name:** Programs Management  
**Purpose:** Organize and manage academic programs  
**Who Uses It:** Administrators and registrars  
**Key Actions Available:**
- View all academic programs
- Create new programs
- Edit program details
- Organize programs by department
- Set program duration and requirements

**What You Need to Know:**
- Programs are the top-level academic structure
- Examples: "Bachelor of Medicine", "Bachelor of Surgery"
- Each program contains multiple courses
- Students are enrolled in programs

## 10. System Settings

**Screen Name:** Admin Settings  
**Purpose:** Configure system-wide options and preferences  
**Who Uses It:** System administrators only  
**Key Actions Available:**
- Configure system settings
- Enable or disable features
- Set default values
- Configure system behavior
- Save changes to settings

**What You Need to Know:**
- Changes here affect the entire system
- Settings are organized by category
- Some settings use toggle switches (on/off)
- Others use text or number inputs
- Changes save automatically in most cases

**Screenshot Reference:** `10_admin_settings.png`

## 11. Syllabus Manager

**Screen Name:** Syllabus Management  
**Purpose:** Organize course content and syllabi  
**Who Uses It:** Administrators and academic coordinators  
**Key Actions Available:**
- View syllabus items organized hierarchically
- Filter by program, period, block, or module
- Create new syllabus items
- Edit existing items
- Reorder items (move up/down)
- Delete items

**What You Need to Know:**
- Syllabus items are organized in a hierarchy:
  - Program (e.g., MBBS)
    - Period (e.g., Year 1, Year 2)
      - Block (e.g., Block 1, Block 2)
        - Module (specific course content)
- Filters help navigate large syllabus structures
- Items can be reordered within the same level

**Screenshot Reference:** `11_syllabus_manager.png`

## Common Actions Across Screens

Most screens share these common elements:

1. **Left Navigation Menu:** Always visible, allows quick navigation
2. **Search/Filter:** Many screens have search to find specific items
3. **Create/Add Button:** Most listing screens have a button to add new items
4. **Edit Actions:** Items in lists can usually be edited
5. **User Profile:** Accessible from most screens (usually top-right)

## Getting Help

If you need help understanding or using any screen:

1. Look for help icons or tooltips on the page
2. Check the system documentation
3. Contact your system administrator
4. Refer to training materials

## Conclusion

These screens provide comprehensive functionality for managing the institution's operations. Each screen is designed for specific tasks and user roles. Understanding what each screen does helps administrators use the system effectively.
